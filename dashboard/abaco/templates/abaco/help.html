    Getting Started with TACC Reactors

    TACC Reactors provide users with a scalable "serverless" computing platform based on Docker container technology and the Actor Model of concurrent computation. The opensource Abaco web service underpins the Reactors service:

    Every TACC rector is associated with a Docker image, and registering a reactor is as simple as specifying a valid image on the Docker Hub. Once registered, a TACC Reactor is assigned an ID and gets a Messages Inbox URL associated with that ID. Sending a POST request to a rector's inbox URL queues a container execution where the message payload of the POST request is injected into the container at runtime.

    The system can handle thousands of concurrent messages to a reactor's inbox; the messages will be queued internally and the containers will be launched sequentially. Thus, reactors can be used to "respond to events" using the webhooks paradigm. Morevover, TACC Reactor containers are authenticated with a short-term access token representing the user who registered the reactor enabling them to access any of TACC's APIs. Additionally, reactor containers have access to other TACC resources such as the user's home directory on the work file system.
